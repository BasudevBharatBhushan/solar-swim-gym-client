# Version 1.2.0

**Release Date:** 2026-03-04

### Added
- **Atomic Payment Workflow**: Implemented a robust atomic payment process via `/pay-invoice` and Supabase RPC `finalize_payment_sync`. This ensures database consistency across transactions, invoices, and subscriptions.
- **Pre-transaction Logging**: Created a pre-transaction record to track payments before external Cayan charges.
- **Auto-void Safeguard**: Implemented an automated Cayan void mechanism in case of sync failures during the database finalization step.
- **Fail Reason Field**: Added explicit failure reason tracking for payment transactions.
- **Manager Passcode Integration**:
  - Added `ManagerPasscodeDialog` to gate sensitive actions.
  - Required manager authorization for subscription lifecycle changes (status, pricing overrides).
  - Required manager authorization for modifying/cancelling invoices and purchases.
- **Manual Payment Support**: 
  - Enabled manual credit card entry in `PaymentDialog` for non-vaulted cards.
  - Integrated field validation and real-time vaulting for new payment methods.
- **Payment Dialog Enhancements**:
  - Detailed tooltips/descriptions for payment buttons:
    - "Pay": Processes full/partial payment and creates records.
    - "Skip": Creates a pending invoice without immediate payment.
    - "Cancel": Returns to the cart without creating any invoice or subscription records.
  - Added "Cancel Payment and Return to Cart" option in the payment dialog.
  - Added instructional message for manual voiding/refunding in invoice cancellation popup.
- **Status Enum Extension**: Added new `SubscriptionStatus` values: `EXPIRED`, `PENDING_PAYMENT`, `PAUSED`.
- **API Testing Suite**: Developed a comprehensive test-api script (`scripts/test-api.ts`) for verifying success, decline, and failure scenarios for the new payment system.

### Changed
- **Deferred Subscription Creation**: Modified checkout logic in `Marketplace.tsx` to delay record creation until the user explicitly clicks "Pay" or "Skip" in the dialog.
- **Membership Purchase Rule**: Refined the existing membership check to allow purchases if the current subscription is not `ACTIVE` (e.g. `CANCELLED`, `PAUSED`).
- **Billing Table Views Refinement**:
  - Integrated `invoice_no` display with fallback support to `invoice_id`.
  - Implemented client-side sorting using `TableSortLabel` for Global Invoices and Transactions.
  - Harmonized filtering UI with the `Accounts.tsx` layout (Paper-wrapped forms, no debounce).
- **Manage Subscription UI**:
  - Improved layout with lighter headers and better spacing.
  - Enhanced details display for active/cancelled subscriptions.
  - Disabled auto-renew toggle with a "Coming Soon" note.
  - Hidden the "Manage" button for cancelled subscriptions.
- **Status Filter Logic**: Updated the "INACTIVE" subscription filter to "PENDING" and adjusted visibility to include all subscriptions that are neither `ACTIVE` nor `CANCELLED`.

### Fixed
- **Double Payment Issue**: Investigated and fixed race conditions/state staleness that could cause redundant `payInvoice` calls.
- **Button Enablement**: Resolved logic error where the "Pay" button remained disabled after backend changes to the payment flow.
